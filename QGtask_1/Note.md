## 坐标系投影和夹角

我们知道向量间的夹角可以用数量积来求
$$
cos\theta = \frac{v_a\cdot v_b}{|v_a|\cdot|v_b|}
$$
用反三角函数可以得弧度

而投影是用cos值乘向量模长获得
$$
projection=|v_a|\cdot cos\theta
$$
所以这两个功能可以共用一个求夹角cos值的函数

##### ~~乱输入的拖走~~

如果输入中有0向量，cos值的计算可能报错，所以多写一步判断

```python
    def __axis_cos_angle(self, vec):
        angles = []
        for axis in self.ori_axis:
            axis_dot = np.dot(vec, axis)
            norm_sum = np.linalg.norm(axis) * np.linalg.norm(vec)
            if norm_sum == 0:
                res = 0
            else:
                res = axis_dot / norm_sum
            angles.append(res)
        return angles
    def _axis_angle(self, vec):
        angles = self.__axis_cos_angle(vec)
        res = []
        for angle in angles:
            res.append(math.acos(angle))
        return angles
    def _axis_projection(self, vec):
        res = []
        norm_v = np.linalg.norm(vec)
        for angle in self.__axis_cos_angle(vec):
            res.append(norm_v * angle)
        return res
```

## 坐标系转换

### 思路

假设我们现在有一个坐标系和一个用该坐标系表示的向量
$$
坐标系=(x轴向量,y轴向量,z轴向量)
初始向量 =(a,b,c)
$$

##### 而每个轴向量又可以表达为

$$
x=(x_1,x_2,x_3)\\y=(y_1,y_2,y_3)\\z=(z_1,z_2,z_3)
$$

##### 那他的真实位置不就是：

$$
ax+by+cz-->
(ax_1+by_1+cz_1,ax_2+by_2+cz_2,ax_3+by_3+cz_3)
$$

然后发现，这玩意的结果跟**乘积**很像；(注意坐标系要表示成列向量)

而我们的目标是将向量从坐标系$sys(x,y,z)$转移到坐标系$sys(x_0,y_0,z_0)$

也就是说，我们要找一个向量$(a_0,b_0,c_0)$使其与坐标系$sys(x_0,y_0,z_0)$算出来的**真实位置**相同

##### 于是我们得到以下等式：

$$
sys(x,y,z)\times初始向量=sys(x_0,y_0,z_0)\times目标向量\\
目标向量=(sys(x,y,z)\times初始向量) sys(x_0,y_0,z_0)^{-1}
$$

既然我们发现了这实际上是矩阵乘积，那处理除法时就也要按矩阵的规矩来

不过`numpy`有自带的逆矩阵转换，直接用就好了

```python
self.vector = np.linalg.inv(self.target) @ (self.basic @ self.vector)
```

这样很好，啊不对啊，我们将样例输入，好像结果不兑啊，~~废了废了~~

这里出现了个新问题，我们是根据比例来算的，那模长怎么办？

##### 比例换模长

我们现在算出来的向量还不是表示在这个坐标系上的具体坐标，而是表示每个轴向量的个数
$$
目标=(xe_x,ye_y,ze_z)\quad但\quad 现在=(x,y,z)
$$
我们需要将每个向量的模求出来，再乘积，这样才是真正想要的坐标

##### 注意

突然发现原本的向量表示还要先转换成**比例系数**，所以一开始得到的向量还要先除以各基向量的模长得比例

## 思考题

1.对于有向图，有权图，公式哪些参数需要改变？

- 无向图->有向图：
  - 边（u，v）从无序变有序，表示u指向v
  - 每个点有了入度和出度，就是被指和指别人的次数
- 无权图->有权图：
  - 每一条边都有一个权重值（u，v，w）
  - 计算路径长度的时候需要考虑权重
- 公式参数：
  - 第一个公式中的求加权系数，在无向无权的情况下，权重默认为1，有边的情况下其实就是$\frac{1}{连接边数}$
    无边情况下为0
  - 第二个公式更新智能体的状态值：$x_i^{(t+1)}=\sum^N_{j=1}{w_{ij}x_{j}^{(t)}}$
    将相连的点加权平均，然后迭代，最后状态值趋于一致
  - 改成有权图后，第一个公式的a需要改变，从{0，1}到{0，1，2……N}
    第二个公式没有变化
  - 改成有向图后，$a_ij \not= a_ji$，同时会影响到$w$的计算

4.理解离散时间和连续时间下算法的异同

- 区别：离散时间在离散的时间点上取值，连续时间在连续的时间范围内取值
  - 连续的时间可以用微分来描述瞬时变化
  - 离散的时间通过差分方程描述状态的步进更新
  - 差分方程：
    - 设有序列$f(k)$,$f(k+2)f(k+1)f(k-1)f(k-2)$为$f(k)$的位移序列
    - 差分运算：
    - 一阶后向差分$\nabla f(k)=f(k)-f(k-1)$   **常用**
